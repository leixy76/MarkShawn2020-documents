## 二、Level 1：先从“不动代码”开始，感受魔法

在咱们一头扎进那个看起来有点“硬核”的黑色窗口之前，我们先来点轻松的。对于绝大多数只想快速把想法变成现实的朋友，完全可以先从下面这些“可视化魔法”开始。

这个阶段的目标是：**建立一个具像化的感知**。让你明白，所谓的“Vibe Coding”到底是一种怎样的体验。

### 如果你想快速生成一个网站或App…

*   **[Lovable](https://lovable.dev/)**：把它想象成一位“AI 产品经理 + 设计师 + 工程师”三合一的伙伴。你只需要用大白话和它聊天，告诉它“我想做一个给独立摄影师展示作品的网站，风格要简约、高级，色调是黑白的”，几分钟后，一个精美的网站雏形就诞生了。
*   **Figma + AI 插件**：Figma 是设计师常用的工具，就像一个数字画板。现在，通过安装 [Anima](https://www.animaapp.com/) 这类 AI 插件，你可以把画好的设计图直接“摇身一变”，转换成真实的网页代码。虽然代码不一定完美，但用于快速演示想法，简直是神器。
*   **网页版 Claude / ChatGPT**：你甚至不需要专门的工具。直接打开这些AI的聊天网页，对它说：“帮我写一个网页代码，功能是有一个输入框和一个按钮，输入任务后，点击按钮可以添加到下方的列表中”，它会立刻给你一段可以直接运行的代码。

### 如果你看到一个很酷的网站想“复刻”一个…

*   **[same.new](https://same.new/)** 或 **Heyboss.ai**：这两个工具就像拥有“写轮眼”的复制大师。你看到一个喜欢的网站，把网址丢给它，它就能分析这个网站的布局、颜色、字体，然后用当前最流行的技术帮你生成一个几乎一模一样的项目框架。这对于学习和借鉴来说，简直不要太方便。

花上一个小时体验一下这些工具，你会发现，原来“创造”的门槛已经如此之低。当你心中那句“哇，原来这样也行！”油然而生时，恭喜你，你已经成功领悟了“Vibe”的精髓。

如果你对此已经心满意足，那么恭喜你，你已经掌握了快速实现想法的捷径。但如果你和我一样，对“幕后”的世界更好奇，渴望更强的掌控力和更专业的体验，那么，欢迎来到 Level 2。

## 三、起飞前的准备：揭开命令行的神秘面纱

从这里开始，我们将要接触到一个新朋友——**命令行**。

没错，就是那个在电影里黑客“噼里啪啦”敲代码的黑色窗口。请放下你所有的心理包袱，它一点也不可怕。

```ad-tip 到底什么是命令行（Command Line）？

把它想象成与电脑的“微信聊天框”。
- **图形界面（我们平时用的）**：就像 App，你通过点击图标、拖动窗口来“告诉”电脑做什么。这很直观，但操作比较慢，而且很多高级功能没有按钮。
- **命令行界面（CLI）**：就像聊天框，你通过输入文字指令来“告诉”电脑做什么。这更直接、更高效，就像给朋友发文字消息，而不是一步步操作图形界面去发送。
几乎所有针对开发者的高级工具，都会提供命令行版本，因为这是最高效的交互方式。
```

我们要做的，就是把这个默认朴素的“聊天框”，打造成一个功能强大、用起来得心应手的“超级控制台”。为此，我们需要先安装一个“管家”。

### 认识你的第一位开发者伙伴：Homebrew

在 Mac 上，开发者们几乎都用 Homebrew 来安装和管理软件。

```ad-tip Homebrew：开发者的“App Store”

你可以把 Homebrew 理解成一个专门给开发者准备的 App Store。
我们平时安装微信、QQ，会去 Mac App Store。而安装开发工具，比如我们后面要用到的各种软件，用 Homebrew 这个“应用商店”来安装和卸载会非常方便，只需要一行命令。
```

**安装步骤**：

1.  **打开“终端”**：在你的 Mac 上，按 `Command + 空格键` 调出聚焦搜索，输入“**终端**” (或 "Terminal")，然后按回车。那个熟悉的“黑窗口”就出现了。

2.  **复制并粘贴命令**：复制下面这整行命令，粘贴到终端里，然后按回车。

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    *   **小贴士**：粘贴时，终端可能会提示你输入电脑的开机密码。放心输入即可，输入时屏幕上不会显示任何字符，这是正常的安全设置。输完后按回车。

3.  **耐心等待**：这个过程会从网上下载并安装 Homebrew，根据你的网络情况，可能需要几分钟到十几分钟。请坐和放宽，泡杯咖啡。当它运行完毕，终端会再次出现一个提示符（通常以 `$` 或 `%` 结尾），说明安装成功了。

好了，我们的“管家”已经就位。现在，真正的冒险要开始了！

## 四、一步步搭建你的 AI 编程“驾驶舱”

接下来，我们将像组装乐高一样，一步步把我们的开发环境搭建起来。请完全放松，只需要复制、粘贴、回车，并理解每一步是为什么。

### 第 1 步：让你的电脑拥有“任意门” (网络代理)

由于我们后面要连接的 Claude、OpenAI 等顶级 AI 服务，它们的服务器都在国外，国内直接访问可能会很慢或无法访问。因此，一个稳定的网络代理是我们的“任意门”，能让我们顺畅地连接到全球的优质资源。

1.  **准备代理工具**：你需要一个代理客户端，比如 ClashX。这里我们不展开如何获取和配置它，假设你已经拥有一个可以正常工作的代理服务。

2.  **告诉终端使用“任意门”**：打开你的代理客户端，并记下它的 HTTP 代理端口号（ClashX 默认为 `7890`）。然后，在终端里执行以下两条命令，告诉所有命令行工具，上网时要走这条“特殊通道”。

    ```bash
    echo 'export ALL_PROXY="http://127.0.0.1:7890"' >> ~/.zshrc
    echo 'export HTTPS_PROXY="http://127.0.0.1:7890"' >> ~/.zshrc
    ```

3.  **让设置生效**：执行 `source ~/.zshrc` 命令，让刚刚的配置立刻生效。

这一步至关重要，是后续所有网络操作能顺利进行的前提。

### 第 2 步：给“黑窗口”换上豪华精装 (iTerm2 + Oh My Zsh)

Mac 自带的终端有点简陋，我们要给它升个级。

*   **iTerm2**：一个更强大、更漂亮的终端替代品。
*   **Oh My Zsh**：一个能让你的终端更好看、更好用的“主题美化+功能增强”包。

**安装步骤**：

1.  **安装 iTerm2**：在终端里输入并执行：
    ```bash
    brew install --cask iterm2
    ```
    安装完成后，在你的“应用程序”里就能找到 iTerm2。**从现在开始，请告别系统自带的终端，始终使用 iTerm2**。

2.  **安装 Oh My Zsh**：打开 iTerm2，复制粘贴下面的命令并执行：
    ```bash
    sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
    ```
    安装成功后，你会发现终端的界面立刻变得不一样了，提示符可能有了颜色，变得更酷了。这就是 Oh My Zsh 的魔力！它还带来了诸如“自动补全命令”、“命令语法高亮”等超级实用的功能。

### 第 3 步：像高手一样管理窗口 (Tmux)

当你以后需要同时处理好几个任务时，比如一边运行程序，一边看日志，一边写命令，`tmux` 就能派上用场了。

```ad-tip Tmux：终端里的“分屏大师”

`tmux` 就像给你的 iTerm2 窗口增加了“多桌面”和“分屏”功能。你可以在一个窗口里开好几个独立的“小窗格”，每个都能独立工作。
它最神奇的地方在于**会话保持**。就算你不小心关了 iTerm2 窗口，或者网络断了，在 `tmux` 里运行的任务也会在后台继续跑。下次回来，还能恢复到之前的工作现场。
```

**安装步骤**：

1.  **安装 tmux**：在 iTerm2 里执行：
    ```bash
    brew install tmux
    ```2.  **美化 tmux (可选，但强烈推荐)**：默认的 tmux 有点丑，我们用一个叫 `oh-my-tmux` 的配置包来美化它。依次执行以下命令：
    ```bash
    cd
    git clone https://github.com/gpakosz/.tmux.git
    ln -s -f .tmux/.tmux.conf
    cp .tmux/.tmux.conf.local .
    ```

现在，在 iTerm2 里输入 `tmux` 并回车，你就进入了一个全新的、被美化过的 tmux 工作区。底部会出现一个状态栏。你可以暂时不用深入学习它的快捷键，只需要知道有这么个“神器”就行。想退出时，按 `Ctrl + d`。

### 第 4 步：前端开发三剑客 (Node.js + nvm + pnpm)

这是网页开发的基础。别担心，我们一行代码都还没开始写。

*   **Node.js**：一个能让你在电脑上（而不仅仅是浏览器里）运行 JavaScript 代码的环境。
*   **nvm** (Node Version Manager)：因为不同项目可能需要不同版本的 Node.js，nvm 就是一个帮你管理这些版本的工具。
*   **pnpm**：一个安装项目依赖（可以理解为插件或扩展包）的工具，它比传统的 `npm` 更快、更节省电脑空间。

**安装步骤**：

1.  **安装 nvm**：在 iTerm2 里执行：
    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
    ```
    安装后，**完全关闭并重新打开 iTerm2**，让 nvm 生效。

2.  **用 nvm 安装 Node.js**：
    ```bash
    nvm install --lts # 安装最新稳定版的 Node.js
    ```
3.  **安装 pnpm**：
    ```bash
    npm install -g pnpm
    ```

至此，我们开发网页所需的基础环境已经全部就绪。

### 第 5 步：为终端注入灵魂：连接 AI 大脑

最激动人心的时刻到了！我们要把全球最顶尖的 AI 编码助手，请进我们的命令行里。

#### 主力选手：Claude Code

这是 Anthropic 公司出品的官方命令行 AI 助手，非常强大。

1.  **安装**：在 iTerm2 中执行：
    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
2.  **登录授权**：
    *   **关键一步**：在进行登录前，**请确保你的网络代理客户端已切换到“美国”节点**。这是 Claude 官方的限制。
    *   在 iTerm2 里，随便进入一个文件夹（比如 `cd ~/Desktop`），然后输入 `claude` 并回车。
    *   它会给你一个网址，并让你在浏览器中打开。在浏览器里登录你的 Claude 账户（需要 Pro 或 Max 订阅）并授权。
    *   授权成功后，回到 iTerm2，它会显示登录成功。

#### 备用选手：OpenAI (Codex/GPT-4o)

我们也把 OpenAI 的能力配置好，以备不时之需。

1.  **获取 API 密钥**：登录 [OpenAI Platform](https://platform.openai.com/api-keys)，创建一个新的 API Key，并复制它。这个 Key 就像是你使用 OpenAI 服务的密码，请妥善保管。

2.  **配置密钥**：在 iTerm2 里执行下面这行命令，记得把你自己的 Key 替换进去。
    ```bash
    echo 'export OPENAI_API_KEY="sk-YourOpenAISecretKey"' >> ~/.zshrc
    ```
3.  **让设置生效**：执行 `source ~/.zshrc`。

现在，我们的命令行已经同时连接了 Claude 和 OpenAI 两个强大的 AI 大脑！

### 第 6 步：最后的拼图：代码编辑器 VSCode

AI 帮我们写完代码后，我们总得有个地方看看吧？VSCode 就是目前最主流、最好用的免费代码编辑器。

1.  **安装 VSCode**：在 iTerm2 里执行：
    ```bash
    brew install --cask visual-studio-code
    ```
2.  **安装推荐插件**：打开安装好的 VSCode，点击左侧的“扩展”图标（像四个方块），搜索并安装以下几个插件，它们会让你的体验更好：
    *   **Chinese (Simplified) Language Pack**：官方中文语言包。
    *   **GitLens**：代码版本管理神器。
    *   **ESLint / Prettier**：帮你保持代码整洁漂亮。

## 五、第一次亲密接触：让 AI 为你写代码

环境搭好了，是时候来一次真正的“Vibe Coding”了。我们将用一分钟时间，从零开始创建一个网页应用。

### 1. 创建项目

我们将使用一个叫 `Vite` 的工具快速创建项目。

1.  在 iTerm2 里，先进入桌面目录：`cd ~/Desktop`
2.  执行创建命令：
    ```bash
    pnpm create vite@latest my-first-app --template react-ts
    ```
    这条命令的意思是：用 pnpm，调用 vite，创建一个叫 `my-first-app` 的新项目，使用 `react-ts` 这个模板。
3.  进入项目并安装依赖：
    ```bash
    cd my-first-app
    pnpm install
    ```
4.  启动项目！
    ```bash
    pnpm run dev
    ```
    终端会显示一个网址，类似 `http://localhost:5173`。按住 `Command` 键并点击这个链接，浏览器就会打开一个漂亮的欢迎页面。我们的第一个应用，已经跑起来了！

### 2. 对 AI 下达你的第一个“Vibe”

现在，我们让 AI 来帮我们写一个新功能：一个随机笑话生成器。

1.  **保持项目运行**，不要关闭刚才的 iTerm2 窗口。按 `Command + T` 新建一个 iTerm2 标签页，它会自动进入同一个项目目录。

2.  **召唤 Claude**：在这个新标签页里，输入 `claude` 并回车。

3.  **下达指令**：现在，到了最关键的时刻。用大白话，清晰地告诉它你想要什么。复制下面这段话，粘贴到 Claude 的交互界面里：

    > 帮我在 `src/components` 目录下，创建一个新的 React 组件，文件名叫 `JokeGenerator.tsx`。
    > 这个组件要实现一个功能：
    > 1.  页面上有一个按钮，文字是“给我讲个笑话”。
    > 2.  当用户点击这个按钮时，从这个网址 `https://official-joke-api.appspot.com/random_joke` 获取一个笑话数据。
    > 3.  把获取到的笑话的“setup”（铺垫）和“punchline”（笑点）显示在按钮下方。
    > 4.  在获取数据时，要显示“加载中...”，如果获取失败，要显示错误信息。
    > 最后，请帮我把这个新组件，添加到主页面 `App.tsx` 里，让它显示出来。

4.  **授权执行**：Claude 会分析你的话，然后告诉你它打算做什么（创建目录、创建文件、写代码、修改主页）。你看完后，输入 `y` (yes) 并回车。

魔法发生了。Claude 会在几秒钟内完成所有编码工作。

5.  **见证奇迹**：回到你之前打开的浏览器页面，刷新一下。你会发现，页面上已经出现了“给我讲个笑话”的按钮！点击它，它真的会去获取并显示笑话！

你，在没有写一行代码的情况下，仅仅通过描述你的“Vibe”，就完成了一个功能的开发。
