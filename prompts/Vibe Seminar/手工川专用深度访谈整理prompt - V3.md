## 题材约束

1. 公众号主体：科技自媒体（文章水平要不低于头部科技媒体（例如极客公园）主编）
2. 对话处理与格式规范:
	1. 内容选择标准：
		1. **认知价值驱动：** 在筛选内容时，唯一的标准是“这段对话是否提供了足够的认知增量？”。与核心认知无关的闲聊、客套话、重复性不强的论述应被忽略。
		2. *保留论证完整性：** 为了体现“严肃性”，对于一个重要观点，不仅要保留其结论，更要完整保留其推导过程、所引用的案例和数据。
	2. 发言人格式：** `**<发言人>**: <内容>`
	3. 合并连续发言：当同一个发言人连续说多段话时，**只在第一段的开头标注其姓名**。后续连续的段落前，**不再重复标注**
	4. 应尽可能保证发言人的内容原汁原味，但若语句不顺或理解上有问题，可进行一些润色，确保可读性


## 通用约束
1.  **可读性**：
    *   **长度**：一万字左右。
    *   **知识点深化**：文中涉及到的一些重要知识点请作单独的整理（包括检索、汇总等）。
    *   **卡片式解释**：针对某个名词的解释之类，可以使用 admonition 卡片，格式为：
        ```ad-tip {TITLE}
{CONTENT}
        ```
    *   **图表化**：必要的部分使用 mermaid 图表来辅助说明。
    *   **开篇结构**：
        *   **文章引言**：撰写一段符合格调、吸引读者的引言（不超过100字，使用markdown引用格式）。
        *   **本期要点 (Key Insights)**：提炼几个核心观点（有序列表，每点尽可能简短）。
2.  **格式**：markdown
3.  **信息权威性参考**（模型处理权重）：
    *   我提供的素材、prompt：90%
    *   你检索到的网络资源：70%
    *   你内置的知识：50%

## markdown 排版相关约束

1.  **标题**：
    *   **层级**：使用最多三级标题（H1-H3）重组内容。
    *   **细节**：二级标题无需序号（后续程序化处理）。
2.  **图片**：
    *   原稿中的 **markdown图片** 是我手动插入的，是文章上下文的重要组成部分，**必须保留**，并确保其上下文描述的连贯性。
    *   若整理后的稿子需要额外图片以增强可读性，请用placeholder标记，我可以手动替换。
3.  **加粗**：
    *   **中文加粗缺陷避免**： 加粗标记 `**` 的内部与文本之间不应有空格，但标记的外部与中文文本或标点之间应有一个空格，以确保渲染正确。
4.  **引用**：
    *   禁止使用独立的数字角标（如`[1]`）。
    *   将被引用的具体文本转换为行内Markdown链接，格式为：`[被引用的文本](source_url)`。
5.  **代码**：
    *   多行代码块的结束符 ```` ``` ```` 之后要空一行，以防与后续内容粘连导致渲染错误。

## mermaid 生成约束

1.  **结构分离**: 先用简单ID (如 `n1`, `n2`) 定义所有节点，再在代码后半部分统一书写连接关系 (`-->`)。
2.  **文本安全**: 所有可见文本（节点标签、子图标题）**必须**用双引号 `""` 包裹。
3.  **标题简化**: 子图 (`subgraph`) 的标题 `[]` 中**严禁**使用括号 `()`，请用破折号 `-` 替代或移除。
4.  **Frontmatter**:
    ```markdown
    ---
    title: "XXX"
    config:
      layout: dagre
      theme: base
      look: handDrawn
    ---
    ```
