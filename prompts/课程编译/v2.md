## markdown 排版相关约束

1.  **标题**：
    *   **层级**：使用最多三级标题（H1-H3）重组内容。
    *   **细节**：二级标题无需序号（后续程序化处理）。
2.  **图片**：
    *   原稿中的 **markdown图片** 是我手动插入的，是文章上下文的重要组成部分，**必须保留**，并确保其上下文描述的连贯性。
    *   若整理后的稿子需要额外图片以增强可读性，请用placeholder标记，我可以手动替换。
3.  **加粗**：
    *   **中文加粗缺陷避免**： 加粗标记 `**` 的内部与文本之间不应有空格，但标记的外部与中文文本或标点之间应有一个空格，以确保渲染正确。
4.  **引用**：
    *   禁止使用独立的数字角标（如`[1]`）。
    *   将被引用的具体文本转换为行内Markdown链接，格式为：`[被引用的文本](source_url)`。
5.  **代码**：
    *   多行代码块的结束符 ```` ``` ```` 之后要空一行，以防与后续内容粘连导致渲染错误。

## mermaid 生成约束

1.  **结构分离**: 先用简单ID (如 `n1`, `n2`) 定义所有节点，再在代码后半部分统一书写连接关系 (`-->`)。
2.  **文本安全**: 所有可见文本（节点标签、子图标题）**必须**用双引号 `""` 包裹。
3.  **标题简化**: 子图 (`subgraph`) 的标题 `[]` 中**严禁**使用括号 `()`，请用破折号 `-` 替代或移除。
4.  **Frontmatter**:
    ```markdown
    ---
    title: "XXX"
    config:
      layout: dagre
      theme: base
      look: handDrawn
    ---
    ```
