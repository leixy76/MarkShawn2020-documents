## 楔子：昨夜无眠，AI 编程圈的“惊天阳谋”

昨夜，Google 开源了 Gemini CLI，我像所有 AI 信徒一样，在第一时间冲了进去。

但短短几个小时，我的心情就像坐了一趟过山车：从兴奋、错愕、震惊，到最后，是一种复杂的敬佩。

我原以为这只是一个新工具的发布，一场神仙打架的序幕。直到我一行行地对比代码，才惊觉：**Google 似乎将闭源的王者 Claude Code ‘逆向’后，直接封装成了一个开源的 Gemini CLI**。

这究竟是彻头彻尾的抄袭，还是对开源精神的伟大贡献？

这篇文章，就是我连夜分析的全部心路历程。它关乎我们每个人的饭碗，也预示着 **Vibe Coding** 时代的真正到来。

## 第一章：初体验——华丽外衣下的“糙”与“似曾相识”

晚上 10 点 39 分，我终于跑通了 `gemini` 命令。

![](https://i.imgur.com/kP4F2J2.png)

> [截图：Gemini CLI 首次运行的主题选择界面，展示了多彩的 Logo 和选项]

第一印象很 Google，华丽，流畅。内置了十几个主题，从 `ansi light` 到 `xcode light`，看得出是想覆盖各种开发者的偏好。

但，问题很快就来了。

> **我（当时的朋友圈吐槽）**：
>
> **问题 1：** 主题配色有问题。`ansi light`、`google code light`、`xcode light` 在系统暗色模式下，文字直接看不见。小厂就算了，大厂犯这种错误，说明产品团队不够用心。
>
> **问题 2：** 不干活一直退出。甚至连一个 `planning` 提示都不给，然后过了 70 秒直接死掉（其他朋友也有这个问题）。

![](https://i.imgur.com/uF8z014.png)

> [截图：暗色模式下，`ansi-light` 主题导致文字几乎不可见的问题]

**初步结论：版本过于早期。**

就在我以为这只是个匆忙发布的半成品，准备洗洗睡时，我注意到了一些不寻常的地方。它的命名、它的结构……甚至它犯错的方式，为什么和 Claude Code 如此相似？

一个可怕的念头在我脑中浮现。

## 第二章：代码为证——100% 确认的“套壳”与“阳谋”

凌晨 1 点，我睡意全无，开始翻看 Gemini CLI 的 GitHub 仓库和本地代码。

> **我（当时的朋友圈吐槽）**：
>
> 据反馈，命名和 cc 都差不多，应该就是逆向的 cc，然后封装成了 gemini cli！但 gemini 本身是值得期待的！我每天都用 gemini 做研究！

很快，我找到了铁证。

> **我（当时的朋友圈吐槽）**：
>
> 对比了一下别人逆向的 cc 的代码，gemini 就是套的逆向，因为用了 IIFE，IIFE 也就是 js 里的立即执行函数，也就是把目标函数原封不动的抄来执行，正常写自己的代码不会这么写，一般是用于套别人的代码。

> [!ad-tip]
>
> **什么是 IIFE (立即执行函数)？**
>
> `(function() { /* ... a bunch of code ... */ })()`
>
> 你可以把它理解成一个“代码黑盒”。当你想把一大段别人的、可能存在变量冲突的代码直接拿来用，又不想污染自己的作用域时，就会用这种方式包裹起来。在 Gemini CLI 的代码里看到这个，就像在一部新手机里发现了一块印着别家 Logo 的主板。

Woc，实锤了。

那一刻我的心情是震惊和愤怒。Google 这么大的公司，竟然公然逆向一个商业级产品，然后开源出来挤兑对手？这操作也太骚了吧。

> **我（当时的朋友圈吐槽）**：
>
> 我挺震惊 gemini 能如此公然逆向一个商业级产品并开源为自己家产品 pr 的。这是彻彻底底的抄袭与挤兑。

就在我准备发文怒斥这种“不讲武德”的行为时，我看到了它的 `LICENSE` 文件：**Apache 2.0**。

> [!ad-summary]
>
> **Apache 2.0 意味着什么？**
>
> 这是一个极其商业友好的开源协议。你可以免费使用、修改、分发代码，甚至用于商业闭源产品，只需要保留原作者的版权信息和声明你对代码所做的修改。

格局瞬间打开。

Google 这不是简单的抄袭，这是一场精心策划的**“阳谋”**。它在告诉全世界：

1.  Anthropic 的 Claude Code 设计思想（约束、安全、交互）是目前业界顶尖的，我认可。
2.  但它闭源，太封闭了。所以我把它“逆向”出来，用最宽松的 Apache 2.0 协议**开源送给大家**。
3.  最好的 Agent 设计范式，我免费送给大家了。但想用好它，想用上多模态、Google Search 这些我的独门绝技？来我的 Gemini 平台，加入我的生态吧。

这不是抄袭，这是**釜底抽薪**。Google 不屑于做一个更好的工具，它要直接定义这个赛道的开源标准。

## 第三章：以子之矛，攻子之盾——我用 Claude Code 修复了 Gemini CLI

凌晨 3 点，最戏剧性的一幕发生了。

我定位到了之前 Gemini CLI 频繁退出的 Bug，发现问题根源和 Claude Code 的一个内部机制极其相似。凭借着我对 Claude Code 逆向代码的熟悉，我直接在 Gemini CLI 的 GitHub 上提了一个 issue，并给出了解决方案。

> **我（当时的朋友圈吐槽）**：
>
> 🎉🎉🎉Gemini Cli 开源发布 2 小时内，我完全基于 claude code 修复了一个它的 bug！！！
> 大陆的朋友用 gemini cli 每分钟会自动退出，请参考我的标准解决方案：https://github.com/google-gemini/gemini-cli/issues/1425#issuecomment-3005831312
> 大家快去点赞支持一波！！！这是 Vibe Coding 革命的胜利！！！

![](https://i.imgur.com/83pQ2gX.png)

> [截图：我在 Gemini CLI 的 GitHub Issue #1425 下的评论，指出问题并提供了解决方案]

这一刻，我深刻地体会到，当一个产品的设计思想足够优秀时，它本身就成了可以跨越产品的“源代码”。**我用“原厂的设计图”，修好了“副厂的流水线”**。

这简直太魔幻了。

## 第四章：重新审视：当艺术品被做成了“乐高套件”

天亮了，我重新梳理了思路。现在，我们不能再用“原创”和“抄袭”的眼光来看待这两个产品了。它们代表了两种截然不同的哲学。

| 维度           | Claude Code (精雕细琢的艺术品)          | Gemini CLI (开源的乐高套件)                  | 南川的观察                                                                  |
| :------------- | :-------------------------------------- | :------------------------------------------- | :-------------------------------------------------------------------------- |
| **设计哲学**   | **约束即自由**，精专主义                | **开放与集成**，生态主义                     | 一个是 iOS，一个是 Android                                                  |
| **代码质量**   | 极高，遵循项目规范，风格一致            | 不错，但核心逻辑是“黑盒”，依赖开源贡献者完善 | Claude Code 像高级工程师，Gemini CLI 像能力全面的实习生，但背后站着整个社区 |
| **安全性**     | **顶尖**，内置 7 重安全防护，刻在骨子里 | **良好**，提供沙盒环境，但安全意识需用户引导 | 在生产环境处理核心代码，我依然首选 Claude 的安全感                          |
| **上下文管理** | **8 段式智能压缩**，记忆力惊人          | 依赖模型原生能力，有 `save_memory` 工具辅助  | 长时间复杂任务，Claude Code 的“不失忆”是巨大优势                            |
| **交互体验**   | 极简、精准，**4 行回复是神来之笔**      | 灵活，信息量大，**多模态交互是王牌**         | 日常使用 Claude Code 更清爽，但 Gemini CLI 更炫酷，更有未来感               |
| **扩展性**     | 有限，依赖官方更新和 MCP                | **无限**，完全开源，可自由定制和扩展         | 这是两者最根本的区别，也是 Google 的阳谋所在                                |
| **杀手锏**     | 深度代码理解与重构、极致的安全感        | **多模态输入、Google 生态、完全开源**        | 一个是“编程宗师”，一个是“创客空间”                                          |

### 我的新选型策略

1.  **当你的任务是“深度耕耘”时，用 Claude Code。**

    - 场景：核心生产项目、大型代码库重构、安全审计、寻找深层 Bug。
    - 你需要的是一个绝对可靠、能深度理解你代码、帮你做出正确决策的“技术合伙人”。

2.  **当你的任务是“探索创造”时，用 Gemini CLI。**

    - 场景：新项目原型验证、个人项目、学习探索、多模态开发（比如把 UI 设计图直接生成代码）。
    - 你需要的是一个创意无限、高度可定制、能连接万物的“超级玩具”和“创客平台”。

3.  **创业机会？**
    - > 另外一个思路，基于 gemini cli 包一个新的 multi-agent 前端的壳，一个用于写代码、一个用于 review、一个用于 pr 等，这个很有价值！至少值一个天使轮！

## 第五章：这是 Vibe Coding 革命的胜利！

回看这疯狂的一夜，我意识到，这起事件的意义远超两个产品本身。

我将其称为：**Vibe Coding 革命的胜利**。

为什么？

因为一个好的“交互范式”和“设计哲学”是如此重要，以至于行业巨头愿意用近乎“抄袭”的方式将其确立为开源标准。这标志着 AI 编程的竞争，已从单纯的模型能力比拼，**升级到了“工作流定义权”的争夺**。

这对我们程序员意味着什么？

1.  **从“工具使用者”到“思想借鉴者”**：我能用 Claude Code 的思想修复 Gemini CLI 的 Bug，你也能。优秀的设计思想正在成为一种可移植、可复用的核心资产。
2.  **从“代码工人”到“软件设计师”**：AI 正在接管越来越多的重复性编码工作。我们的价值不再是“写得多快”，而是“想得多深”——如何分析问题、如何设计系统、如何与 AI 高效协作。
3.  **从“打工人”到“创业者”**：Gemini CLI 这样的开源平台，等于把一个顶尖团队的设计成果和 Google 的多模态能力免费提供给了你。围绕它做二次开发、做垂直领域的解决方案，从未如此简单。

Google 把牌桌掀了，现在轮到我们开发者来决定游戏的玩法。这是一个混乱但充满机遇的时代。

放下对具体工具的执念，拥抱其背后的设计思想。

**因为 AI 正在帮我们敲下那些重复的代码，而我们，要去设计下一个未来。**
